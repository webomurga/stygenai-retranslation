# Translator Style and Authorship Attribution Analysis – User Guide

This project consists of **two separate but integrated Python packages** designed to analyze and classify translator style in literary translation texts:

1. **Feature Extraction Package**: Reads raw text files and morphological analysis files, then converts each text into numerical **Excel (.xlsx)** tables containing statistical, stylistic, and morphological vectors.
2. **Authorship Attribution Package**: Uses the generated `.xlsx` data and related keyword (`.txt`) lists to run machine learning experiments (SVM, KNN, Random Forest, Burrows’ Delta), producing visual graphs and result reports.

Below is a step-by-step explanation of how the data structure should be organized and how to use the system.

---

# 1. Feature Extraction Package

This module transforms textual properties such as word lengths, sentence-initial word counts, punctuation ratios, Type-Token Ratio (TTR), and morphological root/suffix features into mathematical feature representations.

## What Should the Input Folder Structure Look Like?

The program automatically detects translator (and reference corpus) folders based on naming conventions. It searches for paired directories ending with `_txt` and `_morph`.

Your main data directory (for example `corpus_data/`) and keyword directory (`keywords_data/`) should be organized as follows:

```text
corpus_data/
├── ref_txt/                # Folder containing reference texts
│   ├── chunk_1.txt
│   └── chunk_2.txt
├── ref_morph/              # Morphological analysis outputs of reference texts
│   ├── chunk_1.txt
│   └── chunk_2.txt
├── ny_txt/                 # Translator abbreviated as "ny" (e.g., Nihal Yeğinobalı)
│   └── chunk_3.txt
├── ny_morph/
│   └── chunk_3.txt
├── bd_txt/                 # Another translator following the naming format
├── bd_morph/
└── ...

keywords_data/              # Keyword frequency analysis files
├── kwd_filter.txt          # List of all target keywords to analyze (one per line)
├── ny_pos.txt              # Frequently used (positive) keywords for translator "ny"
├── ny_neg.txt              # Infrequently used (negative) keywords for translator "ny"
├── bd_pos.txt
└── bd_neg.txt
```

*Important: File names inside paired `_txt` and `_morph` directories must match exactly so the system can align them correctly.*

---

## How to Run It

Follow these steps in the terminal (command line):

```bash
# Enter the package directory
cd feature_extraction_package

# Install required libraries (first-time setup)
pip install -r requirements.txt

# Start the analysis
# Adjust the paths according to your local directory structure.
python extract_features.py \
    --corpus-dir ../corpus_data/ \
    --output-dir ./output/ \
    --keywords ../keywords_data/
```

## Output

Inside the directory specified with `--output-dir`, an Excel file will be generated for each translator group.
(Example: `vectors_ref_files.xlsx`, `vectors_ny_files.xlsx`)

---

# 2. Authorship Attribution Package (Translator Classification Analysis)

This package takes the `.xlsx` Excel files generated by the **Feature Extraction Package** together with keyword `.txt` lists, and performs cross-validated statistical analyses, binary/multiclass classification, and sample-size robustness tests.

## What Should the Input Folder Structure Look Like?

The default input directory is the `data/` folder inside this package.
You should copy both the generated Excel files and keyword `.txt` files into this directory:

```text
authorship_attribution_package/data/
├── vectors_ref_files.xlsx   # Generated vector Excel files (Reference corpus)
├── vectors_ny_files.xlsx
├── vectors_bd_files.xlsx
├── ny_pos.txt               # Translator-specific keyword files
├── ny_neg.txt
├── bd_pos.txt
└── bd_neg.txt
```

*(If you want to add or rename a translator, simply update the `TRANSLATORS` dictionary inside `authorship_attribution_package/analysis/config.py`.)*

---

## How to Run It

```bash
# Enter the package directory
cd authorship_attribution_package

# Install required machine learning libraries (first-time setup)
pip install -r requirements.txt

# ---
# 1) RUN ALL TESTS
# ---
python run_analysis.py --all --data-path ./data --output-path ./output

# ---
# 2) RUN SPECIFIC ANALYSES ONLY
# ---

# Binary comparisons:
# Compares each translator individually against the reference corpus. (SVM, KNN, Burrows)
python run_analysis.py --binary

# Multiclass classification:
# Attempts to distinguish all translators/reference texts simultaneously.
python run_analysis.py --multiclass

# Sample-size robustness analysis:
# Starts with 5 chunks and increases the number gradually to measure stability.
python run_analysis.py --sample-size

# ---
# 3) SYSTEM CONFIGURATION
# ---

# To work with specific translators only, use the "--translators" flag.
python run_analysis.py --binary --translators yeginobali suveren

# To use only a specific feature set (e.g., stylistic features) in binary classification:
python run_analysis.py --binary --feature-set stylistic
```

---

## Output

Classification accuracy scores for all analyses will be displayed in the terminal.

The system will also save all outputs inside the directory specified with `--output-path`:

* `binary_classification_results.csv`
* `multiclass_classification_results.csv`
* `sample_size_analysis.png`
* `combined_sample_size.png`

These graphs illustrate how classification performance changes as the number of chunks increases.
